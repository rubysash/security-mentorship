## Core Vocabulary and Concepts

### Cloud Service Models (The Stack)
*   **IaaS (Infrastructure as a Service):** You rent the raw "plumbing" (VMs, storage, network). You manage the OS (e.g., AWS EC2).
*   **PaaS (Platform as a Service):** You rent a platform to run your code. You don't manage the OS (e.g., AWS Elastic Beanstalk, Heroku).
*   **SaaS (Software as a Service):** You use a complete software application over the web (e.g., Office 365, Slack).

### Deployment Models
*   **Public Cloud:** Services offered over the public internet (AWS, Azure, GCP).
*   **Private Cloud:** Services used exclusively by a single organization, often on-premises.
*   **Hybrid Cloud:** A mix of both public and private cloud environments.

### Key Cloud Concepts
*   **Shared Responsibility Model:** The cloud provider secures the "Cloud" (physical security, hardware); YOU secure what's "In the Cloud" (OS, apps, data).
*   **Scalability:** The ability to add resources (e.g., more servers) to handle more load.
*   **Elasticity:** The ability to automatically add OR remove resources based on demand.
*   **High Availability (HA):** Ensuring a system is always accessible, often by spreading it across multiple data centers (Availability Zones).
