# Git Foundations: Version Control

## Objective
Memorize and understand the key concepts and common commands used in modern software development and automation pipelines.

## Legal & Ethical Disclaimer
**FOR EDUCATIONAL USE ONLY.** Never commit or push sensitive data (API keys, passwords, PII) to public repositories. Always use a `.gitignore` file to protect your credentials.

## Prerequisites
- A GitHub account and a desire to track your technical journey.

## Why This Matters
Version control is the lifeblood of DevOps and secure software development. You no longer "copy-paste" code; you commit and push it. This allows for change tracking, collaboration, and automated deployments.

---

## Core Vocabulary and Concepts

### Key Git Terms
*   **Repository (Repo):** A project folder that is being tracked by Git.
*   **Commit:** A "snapshot" of your project at a specific point in time.
*   **Stage:** The process of preparing changes to be committed.
*   **Branch:** A parallel version of a repository (e.g., `main`, `feature-vlan-scan`).
*   **Merge:** The process of combining changes from one branch into another.

### Interaction Terms
*   **Clone:** Copying a repository from a server (GitHub) to your local machine.
*   **Pull:** Fetching and merging changes from a server to your local machine.
*   **Push:** Sending your local commits to the server.
*   **Pull Request (PR):** Proposing your changes to a repository and asking for a review.
