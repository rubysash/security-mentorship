# DevOps Foundations

## Objective
Memorize and understand the fundamental methodologies and automation principles used in modern software delivery.

## Legal & Ethical Disclaimer
**FOR EDUCATIONAL USE ONLY.** DevOps practices involve automation that can deploy real resources (and costs) if not managed correctly. Always practice in a "sandbox" or "dry-run" mode first.

## Core Vocabulary and Concepts

### The DevOps Lifecycle
A continuous loop of planning, coding, building, testing, releasing, deploying, operating, and monitoring.

### CI/CD (Continuous Integration / Continuous Deployment)
*   **Continuous Integration (CI):** Developers push code to a shared repo frequently. Automated tests run to catch bugs early.
*   **Continuous Deployment (CD):** Validated code is automatically pushed to production.
*   **Pipeline:** The automated series of steps (the "conveyor belt") that code goes through to become a running application.

### IaC (Infrastructure as Code)
*   **Concept:** Managing servers and networks using configuration files (code) instead of manual clicks.
*   **Benefits:** Consistency, speed, and the ability to track changes in Git.
*   **Common Tools:** Terraform, Ansible, CloudFormation.

### Microservices and Containers
*   **Microservices:** Breaking a single application into small, independent "services" that communicate over a network.
*   **Containerization:** Packaging an application and its dependencies into a single "container" (e.g., Docker) to run anywhere.
*   **Orchestration:** Managing thousands of containers at scale (e.g., Kubernetes).
