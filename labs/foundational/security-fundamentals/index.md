# Security Foundations

## Objective
Identify and memorize the foundational principles and technical controls used to protect information systems.

## Legal & Ethical Disclaimer
**FOR EDUCATIONAL USE ONLY.** Learning security concepts for the purpose of hacking or unauthorized access is illegal. These skills are to be used only for defensive purposes on systems you own or have permission to test.

## Prerequisites
- Completion of the **Universal IT Foundations: The Common Core** lab.

## Why This Matters
Security is no longer a separate job; it is a feature of every technical role. Whether you're a cloud engineer, a developer, or a sysadmin, you must understand the basics of defense and attack.

---

## Core Vocabulary and Concepts

### The CIA Triad (The Mission)
Every security control must address at least one of these:
*   **Confidentiality:** Ensuring data is only seen by authorized people (Encryption).
*   **Integrity:** Ensuring data has not been modified (Hashing).
*   **Availability:** Ensuring data is accessible when needed (Backups, Redundancy).

### Threat Actors and Malware
*   **Threat Actor:** Any person or entity that poses a risk to an organization (e.g., Insider, Hacktivist, State-Actor).
*   **Social Engineering:** Manipulating people into revealing secrets (Phishing, Vishing).
*   **Malware:** Malicious software (Virus, Worm, Ransomware, Trojan).

### Defensive Controls
*   **Encryption:** Converting data into unreadable text (Symmetric vs. Asymmetric).
*   **Hashing:** Creating a unique digital fingerprint of a file (MD5, SHA-256).
*   **Authentication (AAA):** Identification, Authentication, Authorization, Accounting.
*   **MFA (Multi-Factor Authentication):** Something you KNOW, something you HAVE, something you ARE.
